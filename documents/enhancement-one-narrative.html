Enhancement One: Software Design and Engineering Artifact Description The artifact I selected for this enhancement is my CS 340 Grazioso Salvare dashboard project. This project was originally created in the database course as an interactive dashboard that connects a Python CRUD module to a MongoDB animal shelter database. The dashboard was designed to help Grazioso Salvare identify possible rescue-dog candidates from the Austin Animal Center dataset. The original artifact included a Python CRUD module, a JupyterDash dashboard notebook, and the Grazioso Salvare logo used in the interface. For this enhancement, I focused on improving the software design and engineering quality of the artifact. The original project met the basic assignment requirements, but the enhanced version is cleaner, easier to maintain, and more professional. I improved the CRUD module by adding stronger input validation, better error handling, clearer documentation, and configuration through environment variables. I also improved the dashboard by organizing the rescue filters into reusable data structures, improving the callback logic, adding clearer layout sections, and making the table, chart, and map behavior easier to understand. Justification for Inclusion in the ePortfolio I selected this artifact because it demonstrates several important computer science skills in one project. It shows database access, Python programming, object-oriented design, dashboard development, and user-focused data presentation. This makes it a strong artifact for my ePortfolio because it does more than show code; it shows how code can be used to solve a practical problem for an organization. The dashboard supports decision making by allowing users to filter shelter animals based on rescue-type criteria and view the results in a table, chart, and map. This artifact also showcases my ability to improve existing software. The enhanced CRUD module now has a clearer structure and safer behavior. The create, read, update, delete, and count methods include validation and error handling so the module is more reliable. The dashboard enhancement improves maintainability because the rescue-type filters are stored in one dictionary instead of being scattered throughout the callback logic. This makes the application easier to update if the organization changes its rescue-dog requirements later. The enhanced dashboard also improves the user experience by adding a record count, better filter labels, more consistent layout, and clearer messages when data is unavailable. Course Outcomes This enhancement helped me make progress toward the course outcome about using well-founded techniques, skills, and tools in computing practices to implement computer solutions that deliver value. The enhanced artifact uses Python, MongoDB, Pandas, Dash, Plotly, and Dash Leaflet to support an interactive database-driven application. It also helped me make progress toward the outcome related to professional communication because I had to explain the purpose of the artifact, the design changes, and the value of the enhancement in a way that would make sense to both technical and nontechnical audiences. This enhancement also supports the course outcome related to designing and evaluating computing solutions. I improved the design by separating database operations from dashboard presentation and by reducing repeated logic in the dashboard. I also made progress toward the security mindset outcome by reducing hard-coded configuration through environment variables and by adding safer validation for database operations. I still plan to use later enhancements to more directly demonstrate algorithms and data structures, but this artifact strongly supports the software design and engineering category. Reflection on the Enhancement Process While enhancing this artifact, I learned that improving software is not only about adding new features. A meaningful enhancement can also come from making the code safer, clearer, and easier to maintain. The original dashboard worked, but the enhanced version is better organized and more professional. The biggest improvement was separating responsibilities more clearly. The CRUD module is responsible for database operations, while the dashboard focuses on filtering, displaying, and interacting with the data. One challenge I faced was keeping the enhancement focused on software design and engineering instead of changing the entire project. I wanted the artifact to remain recognizable as the original CS 340 project while still showing real improvement. Another challenge was improving the code without breaking the dashboard behavior required by the original assignment, such as keeping the DataFrame cleanup for the MongoDB _id field and preserving the interactive chart and map features. Overall, this enhancement helped me better understand how database applications can be improved through cleaner design, stronger validation, better organization, and clearer communication.
