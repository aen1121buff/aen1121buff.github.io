Enhancement Two: Algorithms and Data Structure Artifact Description The artifact I selected for this enhancement is my CS 340 Grazioso Salvare dashboard project. This project was originally created in CS 340: Advanced Programming Concepts. It used Python, MongoDB, Dash, Pandas, Plotly, and Dash Leaflet to display animal shelter records from the Austin Animal Center dataset. The dashboard was designed to help Grazioso Salvare identify dogs that may be good candidates for different types of rescue training. The original version included a CRUD module, a MongoDB connection, filter options, an interactive table, a chart, and a map. For Milestone Three, I enhanced this artifact in the category of algorithms and data structure. The original dashboard could filter animal records by rescue type, but it did not rank the results or help the user quickly compare which animals were the strongest candidates. The enhanced version adds a rescue suitability scoring algorithm. This algorithm scores each animal based on breed, age, sex, and outcome type, then sorts the results so the highest scoring rescue candidates appear first in the dashboard table. Justification for Inclusion in the ePortfolio I selected this artifact for my ePortfolio because it shows how a computer science project can solve a practical decision making problem. Grazioso Salvare needs to find dogs that match different rescue training needs, and the dashboard gives users a way to search, filter, view, and compare possible candidates. This makes the artifact stronger than a simple programming exercise because it connects code, data, and user needs in one application. This artifact also fits the algorithms and data structure category because the enhancement adds clear algorithmic logic to the existing dashboard. I created a rescue criteria data structure that stores the breed, age, and sex rules for each rescue type. I also created a scoring function that assigns points to each animal. The ranked records are stored in a list, given a rescue_score value, and sorted in descending order. These changes show my ability to use dictionaries, sets, lists, conditional logic, and sorting to improve an application. The artifact was improved because the dashboard now does more than return matching records. It gives the user ranked results, which makes the output more useful. Instead of manually reading through many rows, the user can start with the animals that best match the rescue training profile. The enhancement also makes the logic easier to update because the rescue criteria are stored in one reusable structure instead of being scattered throughout the code. Course Outcomes This enhancement directly supports Course Outcome 3, which focuses on designing and evaluating computing solutions using algorithmic principles and computer science practices while managing tradeoffs. The scoring algorithm uses rule based logic to evaluate each animal and rank the results. The main tradeoff is that the algorithm is simple and explainable instead of being complex like a machine learning model. I chose this approach because the client criteria are clear, and the user needs results that are easy to understand and justify. The time complexity of the enhancement is mainly O(n log n). Each animal is scored once, which is O(n), and then the scored list is sorted, which is O(n log n). The total complexity is O(n log n) because sorting is the largest part of the process. The space complexity is O(n) because the enhanced version creates a new list of enriched records with a rescue_score field. Reflection on the Enhancement Process While enhancing this artifact, I learned that algorithms do not always have to be complicated to add value. The original dashboard already filtered records, but the user still had to compare many animals manually. Adding a scoring and ranking algorithm made the dashboard more useful without changing the entire purpose of the project. This helped me understand that a good enhancement can improve decision making while still keeping the application simple and maintainable. One challenge I faced was deciding how to score the animals fairly. Breed, age, sex, and outcome type are not equally important, so I had to choose point values that made sense based on the rescue criteria. I gave breed and age the highest value because they are the most important parts of the training profile. I gave smaller values to sex and outcome type because they still matter but should not outweigh the main rescue requirements. Another challenge was balancing simplicity and usefulness. A more advanced model could use more fields, but that would also make the results harder to explain. I chose a rule based algorithm because it is transparent and easier for a reviewer, instructor, or future developer to understand. This was a good tradeoff for this milestone because the goal was to demonstrate algorithms and data structures in a clear, professional, and realistic way. Overall, this enhancement helped me show growth in algorithmic thinking. I improved an existing full stack dashboard by adding scoring, ranking, sorting, and reusable criteria structures. I also learned how to explain the performance of the enhancement using time and space complexity. This makes the artifact a stronger part of my ePortfolio because it now demonstrates both application development and algorithmic problem solving.
