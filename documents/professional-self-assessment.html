Professional Self Assessment My name is Anthony Cheung, and this ePortfolio represents the skills and growth I have developed throughout the Computer Science program at Southern New Hampshire University. Completing the coursework in this program and building this final portfolio helped me connect my classroom learning to real software development practices. Through projects involving software design, databases, algorithms, security, and communication, I have strengthened my ability to design technical solutions that are organized, useful, and built with the needs of users and stakeholders in mind. This ePortfolio shows my progress through a complete enhancement of my CS 340 Grazioso Salvare project, which uses Python, MongoDB, Dash, Pandas, Plotly, and Dash Leaflet to support animal rescue classification and dashboard reporting. Throughout the program, I learned that computer science is not only about writing code. It also requires planning, communication, testing, documentation, and the ability to explain technical choices to different audiences. In courses such as CS 250, I practiced working in an Agile environment and learned how user stories, stakeholder feedback, sprint planning, and product backlogs help guide software development. These skills helped me understand how collaboration supports better decision making. I also learned how to communicate technical information clearly through written reports, code review planning, journal reflections, project documentation, and final narratives. This helped me become more prepared to explain my work to both technical and nontechnical audiences. My work with data structures and algorithms helped me understand how software can be improved through better logic and more efficient design. In my enhanced artifact, I added rescue suitability scoring and ranked sorting to make the dashboard more useful for users who need to identify animals that best match rescue needs. This improvement showed my ability to take raw data, apply decision logic, and present results in a way that supports practical decision making. Instead of only displaying database records, the enhanced project uses algorithmic thinking to organize results by usefulness and priority. Software engineering and database development were major parts of my growth in the program. My CS 340 artifact began as a working dashboard connected to a MongoDB database, but the enhancements made it cleaner, safer, and more professional. I improved the structure of the code, strengthened CRUD operations, improved database filtering, added validation, and made the application easier to maintain. These changes show my ability to build a working software solution while also thinking about long term maintenance and real world use. The database improvements were especially important because they made the project more reliable and helped connect user actions in the dashboard to accurate database results. Security was also an important part of my learning. During the enhancement process, I looked at areas where the original project could be improved, such as hardcoded credentials, weak validation, and unsafe handling of user input. These changes helped me demonstrate a stronger security mindset. I learned that secure software design must be considered throughout the development process, not added only at the end. By improving validation and safer database access, I showed that I can think about privacy, data protection, and possible misuse while building software. The artifacts in this ePortfolio fit together as one complete example of my growth as a computer science student. The original CS 340 dashboard shows where the project started. The enhanced version shows how I improved the same project across software design and engineering, algorithms and data structures, and databases. The code review explains my planning process and identifies areas for improvement. The narratives explain what I changed, why I changed it, what skills I demonstrated, and how the enhancements connect to the computer science program outcomes. Together, these pieces show that I can analyze existing software, improve it with purpose, explain my work clearly, and create a professional portfolio that supports my career goals in computer science.
