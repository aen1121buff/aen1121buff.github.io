Professional Self Assessment

My name is Anthony Cheung, and this ePortfolio represents the skills and growth I have developed throughout the Computer Science program at Southern New Hampshire University. Completing this program and building this final portfolio helped me connect classroom learning to real software development practices. Through projects involving software design, databases, algorithms, security, and communication, I strengthened my ability to design technical solutions that are organized, useful, and built with the needs of users and stakeholders in mind.

The main artifact in this portfolio is my CS 340 Grazioso Salvare animal shelter dashboard. The original project used Python, MongoDB, Dash, Pandas, Plotly, and Dash Leaflet to display animal shelter records from the Austin Animal Center dataset. For CS 499, I enhanced this artifact across three major areas of computer science. These areas are software design and engineering, algorithms and data structures, and databases.

Throughout the program, I learned that computer science is not only about writing code. It also requires planning, communication, testing, documentation, and the ability to explain technical choices to different audiences. In courses such as CS 250, I practiced Agile methods, user stories, stakeholder feedback, sprint planning, and backlog organization. These experiences helped me understand how collaboration supports better software decisions.

My work with data structures and algorithms helped me understand how software can be improved through better logic and more efficient design. In my enhanced artifact, I added rescue suitability scoring and ranked sorting to make the dashboard more useful for users who need to identify animals that best match rescue needs. This improvement showed my ability to take raw data, apply decision logic, and present results in a way that supports practical decision making.

Software engineering and database development were also major parts of my growth. My original CS 340 artifact worked as a database connected dashboard, but the enhancements made it cleaner, safer, and more professional. I improved code structure, database filtering, validation, CRUD operations, and overall maintainability. These changes show that I can improve existing software while thinking about real world use and long term support.

Security was another important part of my learning. During the enhancement process, I reviewed weaknesses such as hardcoded credentials, weak validation, and unsafe handling of database input. I improved validation and database access to show a stronger security mindset. This helped me understand that secure design must be part of the development process from the beginning.

This ePortfolio shows my ability to analyze existing code, improve it with purpose, document my choices, and present my work professionally. The original artifact shows where the project started, and the enhanced artifact shows how I improved it across the major computer science categories.

Enhancement One: Software Design and Engineering

The software design and engineering enhancement focused on improving the structure, readability, maintainability, and usability of the dashboard. The original artifact worked, but there were areas where the code could be organized better and made easier to update. I improved the project layout, cleaned up repeated logic, improved comments, and made the dashboard easier for another developer or stakeholder to understand.

This enhancement demonstrates my ability to analyze existing software and improve it with a clear purpose. I focused on making the dashboard more professional and easier to maintain. I also considered the user experience by improving how information is presented to the user.

Enhancement Two: Algorithms and Data Structures

The algorithms and data structures enhancement focused on improving how the dashboard processes and organizes animal records. I added rescue suitability scoring so that animals could be ranked based on how closely they matched rescue needs. This made the dashboard more useful because it no longer only displayed filtered records. It also helped prioritize the best matches.

This enhancement demonstrates algorithmic thinking because the project uses rules, scoring, sorting, and ranked results to support decision making. The data is organized in a way that makes it easier for users to identify the most relevant animals quickly.

View Algorithms Narrative

Enhancement Three: Databases

The databases enhancement focused on improving the MongoDB connection, CRUD operations, query structure, validation, and database reliability. The original project connected to MongoDB and returned shelter records, but the enhanced version improves how the application validates queries, retrieves filtered records, handles database operations, and supports dashboard performance.

This enhancement is important because the dashboard depends on accurate and reliable database results. Improving database access helped make the project more useful and better prepared for real world use. This enhancement also supports a stronger security mindset by improving validation and safer database interaction.
